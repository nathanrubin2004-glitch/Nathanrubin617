## Interaction Contract

### Chat Bubble (CHAT-01, CHAT-02)
- Position: `fixed; bottom: 2rem; right: 2rem; z-index: 40`
- State: always visible on all 4 pages
- Hover: `transform: scale(1.1)` — existing CSS, no change
- Click: toggles `.active` class on `.chat-window`

### Chat Panel Open/Close (CHAT-02, CHAT-03)
- Open: `.chat-window.active { display: flex }` — toggle via JS
- Close: click "✕" button or re-click bubble
- Note: CSS animation (Phase 4 scope) is NOT added here — plain display toggle only

### Loading / Typing Indicator (CHAT-08)
- HTML structure to inject while awaiting API response:
  ```html
  <div class="chat-message message-assistant" id="typing-indicator">
    <div class="typing-indicator">
      <div class="typing-dot"></div>
      <div class="typing-dot"></div>
      <div class="typing-dot"></div>
    </div>
  </div>
  ```
- CSS already defined in `styles.css` (lines 171–203) — three dots bouncing with `animation: typing 1.4s infinite`
- Show indicator: inject element into `#chat-messages` immediately after user message is added
- Remove indicator: remove the element before inserting the assistant reply bubble
- Auto-scroll applies after indicator is injected (same as message scroll behavior)

### Input Disabled State (CHAT-07)
- `chatSend.disabled = true` + `chatInput.disabled = true` while awaiting API
- Disabled send button: `background: #d1d5db; cursor: not-allowed` — existing CSS
- Re-enable in `finally` block regardless of success/error

### Error State (CHAT-13)
- Error message rendered as a standard assistant message bubble (`.message-assistant`)
- Copy: "Something went wrong — try again"
- Applies to both HTTP error responses and network-level `catch` — single error path

### Auto-scroll (CHAT-11)
- `chatMessages.scrollTop = chatMessages.scrollHeight` after every message append (user, assistant, indicator)

## Mobile Layout Contract

### Full-Width Panel (CHAT-17)

Replace the existing `@media (max-width: 640px)` block in `styles.css` (lines 251–262) with:

```css
@media (max-width: 640px) {
    .chat-window {
        width: 100%;
        max-width: 100%;
        left: 0;
        right: 0;
        bottom: 0;
        height: 70vh;
        border-radius: 12px 12px 0 0;
    }

    .message-bubble {
        max-width: 90%;
    }
}
```

- Edge-to-edge width with no side margins
- Flat bottom edge, 12px rounded top corners — native bottom-sheet feel
- Height: `70vh` (taller than the current 500px fixed value to fill more screen real estate without obscuring page content)
- `left: 0; right: 0` — overrides the desktop `right: 2rem` positioning

### iOS Safe-Area Insets (CHAT-18)

Apply to both the chat panel and the floating bubble so neither overlaps iOS browser chrome:

Chat panel — add to `.chat-input-container` in `styles.css`:
```css
.chat-input-container {
    /* existing properties preserved */
    padding-bottom: calc(1rem + env(safe-area-inset-bottom));
}
```

Chat bubble — update `.chat-bubble` `bottom` value:
```css
.chat-bubble {
    /* existing properties preserved */
    bottom: calc(2rem + env(safe-area-inset-bottom));
}
```

Note: `env(safe-area-inset-bottom)` evaluates to `0` on non-iOS devices — no visual change on desktop.

Requires `<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover">` on all 4 pages to activate safe-area env variables. Verify this meta tag is present.
