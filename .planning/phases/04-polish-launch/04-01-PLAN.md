# Phase 04 Plan 01: Polish & Launch

## Objective

Add starter question chips to the chat widget, add a CSS open/close animation for the chat panel, and verify production security (API key never visible in browser).

## Context

- `chat.js` — shared widget logic (project root)
- `styles.css` — shared styles (project root)
- `index.html`, `book.html`, `basketball.html`, `contact.html` — 4 pages with chat widget

## Tasks

### Task 1: Add starter question chips

Add 3 clickable question chip buttons that appear below the greeting message when the chat panel first opens. Clicking a chip sends that question as if the user typed it. Chips should disappear after the first message is sent.

**Done criteria:**
- 3 chips appear below greeting on first open
- Clicking a chip populates and sends the question
- Chips are styled consistently with the blue color scheme
- Chips do not reappear once a conversation is started

### Task 2: Add CSS open/close animation for chat panel

Replace the `display: none / display: flex` toggle with a CSS animation so the panel slides/fades in and out smoothly. The animation should be subtle and fast (200-300ms).

**Done criteria:**
- Chat panel animates in when opened
- Chat panel animates out when closed
- Animation works on both desktop and mobile
- No layout flash or jump during animation

### Task 3: Production validation checkpoint

Verify that on the live site, the Anthropic API key is never visible in browser DevTools network tab during a real chat session.

**Done criteria:**
- Network tab shows POST to `/.netlify/functions/chat` only
- No `x-api-key` header visible on outbound browser requests
- Response contains `reply` field with assistant message
- Documented in SUMMARY

## Verification

- [ ] 3 starter chips visible below greeting on chat open
- [ ] Clicking a chip sends the question successfully
- [ ] Chips disappear after first user message
- [ ] Chat panel animates open and closed
- [ ] API key not exposed in browser network requests
- [ ] Widget works on all 4 pages

## Success Criteria

1. Three starter question chips appear below the greeting when the chat panel opens; clicking a chip sends that question
2. The chat panel opens and closes with a visible CSS animation
3. No Anthropic API key or system prompt content is visible in browser DevTools network tab during a live chat session
