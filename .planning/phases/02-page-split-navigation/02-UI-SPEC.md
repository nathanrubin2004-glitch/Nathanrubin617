## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, scrollbar thumb radius |
| sm | 8px | Scrollbar width, compact padding |
| md | 16px | Default element padding (px-4, py-2 on nav CTA button) |
| lg | 24px | Nav horizontal padding (px-6), section padding |
| xl | 32px | Mobile menu link padding (px-8), page top margin (mt-8) |
| 2xl | 48px | Section bottom margin (mb-12 = 48px) |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions: Nav touch targets use `py-2` (8px vertical) on desktop links and `py-2` (8px) on mobile links — both exceed the 44px minimum touch target when combined with font size and line height. Mobile hamburger button uses `p-2` (8px) around a 32px icon = 48px touch target, which satisfies the 44px minimum.

## CSS Extraction Contract

### styles.css — what goes in

Extract the verbatim shared `<style>` block from any non-index page (basketball.html lines 15–277 is the canonical source — no page-specific rules). Content includes:

- `:root` CSS variables (`--color-primary`, `--color-primary-dark`, `--color-bg-light`)
- `body` font and background
- `.nav-glass` glassmorphism rule
- `::-webkit-scrollbar` rules (3 selectors)
- All chat widget CSS (placeholder styles — Phase 3 will use them)

### index.html — what stays inline

After extraction, index.html retains a residual `<style>` block with hero-specific rules only:
- `.hero-bg` (background image, overlay)
- `.hero-bg::before` (dark overlay)
- `.social-icons-container` (absolute-positioned social links)
- `.social-icon:hover` (scale transform)
- `.bottom-right-logo` (absolute-positioned logo)

### Load order (critical)

```html
<script src="https://cdn.tailwindcss.com"></script>
<link rel="stylesheet" href="styles.css">
<!-- index.html only: residual <style> block follows here -->
```

`styles.css` MUST be placed after the Tailwind CDN script. Placing it before causes Tailwind base styles to override custom rules.

### styles.css path

File at repo root: `styles.css` (same level as all HTML files). Linked as `href="styles.css"` (relative, no leading slash). Netlify serves from repo root by default — no `publish` directory configured in `netlify.toml`.

## Interaction Contract

### Nav hamburger (mobile)

- Trigger: tap `#mobile-menu-btn` (hamburger icon, `h-8 w-8` SVG)
- Effect: toggle `hidden` class on `#mobile-menu` (existing JS handles this — no changes in Phase 2)
- Touch target: 48px (8px padding around 32px icon) — satisfies 44px minimum
- Active indicator applies to mobile menu links as well as desktop links (Claude's discretion confirmed in CONTEXT.md)

### Nav links (all breakpoints)

- Hover: `hover:text-blue-600` on inactive links — CSS-only, no JS
- Active: `text-blue-600 font-semibold` hardcoded, no JS, no flash of unstyled content
- All hrefs: file-based (`index.html`, `book.html`, `basketball.html`, `contact.html`) — NAV-02 already satisfied
- "Get the Book" href: `book.html#purchase` — anchor preserved, no change

### Sticky nav

- `sticky top-0 z-50` — nav stays visible during scroll on all pages
- Glassmorphism (`rgba(255,255,255,0.9)` + `backdrop-filter: blur(10px)`) — requires styles.css to load correctly

### No new interactions introduced

Phase 2 adds only the active nav indicator (CSS class change) and extracts shared CSS. Zero new JS, zero new interactive components.

## Anti-Patterns (executor must avoid)

| Anti-Pattern | Consequence | Correct Approach |
|-------------|-------------|-----------------|
| Moving `.hero-bg`, `.social-icons-container`, `.bottom-right-logo` to styles.css | Hero background disappears on index.html; social icons layout breaks | Keep those 4 rules inline on index.html only |
| Placing `<link rel="stylesheet" href="styles.css">` before Tailwind CDN `<script>` | Tailwind base styles override custom nav/scrollbar/chat rules | Always: Tailwind `<script>` first, then `<link>` |
| Adding `hover:text-blue-600` to the active nav link | Blue appears twice; no functional difference but is semantically wrong | Active link has only `text-blue-600 font-semibold transition-colors` |
| Using JS to detect active page | Violates D-03; adds JS dependency to navigation | Hardcode active class per file |
| Changing any `href` value during nav edits | Breaks navigation (NAV-01, NAV-02) | Only the `class` attribute changes; hrefs are untouched |
| Placing styles.css in a subdirectory | 404 on Netlify — all custom styles absent, Tailwind utilities still work | styles.css at repo root, same level as HTML files |
